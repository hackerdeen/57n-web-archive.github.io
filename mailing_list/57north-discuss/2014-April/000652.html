Hi,

Not coming in tonight - Got disk problems :-(

I got a Startech 4-port PCIe SATA III controller card (PEXSAT34RH) to
use with some shiny new 4TB SATA disks that I bought. This controller is
based on the Marvell 88SE9230 chipset and, I now discover, lots of
people have been having problems trying to use it under Linux...

To cut a long story short, I thought it was because of the SATA III 6G
speed at first so I set a "libata" module parameter to limit the speed:

  /etc/modprobe.d/libata.conf
  options libata force 3.0Gbps

That did NOT cure the problem :-(

However, the system is a lot more stable when I disable NCQ (Native
Command Queueing) even if I leave it at 6G (i.e. not limited):

  options libata force noncq

Now in the process of restoring ~2TB from backups...
