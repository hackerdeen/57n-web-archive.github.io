Hi,

Another thing me and Rob disagreed about last night was my CRC's ;-)

You might be interested to know that Minix3 still uses the same 1986
vintage code that I use:

  http://www.minix3.org/iso.previous/Intel-2.0.3/crc.c

This calculates a 16-bit 'checksum' based on a polynomial and a table of
feedback terms that was originally used for XYZmodem error checking.

I also use the file length as well as it's CRC to create a digital
signature. Maybe we can do some bit flipping experiments to see if I can
detect bit-rot in my files?
