I had to change the default to "current" for "git" version 1.7.9.5
under Ubuntu 12.04 LTS. I also discovered that "git-sendmail" is not
installed by default under Ubuntu 12.04 LTS when you install "git":

> ajt at beluga:~$ vim /home/ajt/.gitconfi ... [push] default = simple
> 
> [sendemail] smtpencryption = tls smtpserver = mail.ukfsn.org 
> smtpuser = tony.travis at minke-informatics.co.uk smtpserverport =
> 587 confirm = auto ... mkdir src/57North cd src/57North git clone
> http://git.57north.co/git/calendar.git Cloning into 'calendar'... 
> error: Malformed value for push.default: simple error: Must be one
> of nothing, matching, tracking or current. fatal: bad config file
> line 7 in /home/ajt/.gitconfig ... man git-config
> 
> # change default for git version 1.7.9.5 vim /home/ajt/.gitconfig 
> ... [push] default = current ... git config --global
> sendemail.confirm auto git clone
> http://git.57north.co/git/calendar.git cd calendar vim
> calendar.57northmembersgit add calendar.57northmembers git commit
> -m "Added Tony Travis's birthday" git send-email --to jones at sdf.org
> -1 ... git: 'send-email' is not a git command. See 'git --help'.
