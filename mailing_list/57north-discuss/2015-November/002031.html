Hey all,

I just uploaded a pile of code to https://github.com/hackerdeen/57n-id
containing a web-based single sign-on system. It backs off an LDAP
database for its account database, and contains rudimentary account
administration functionality, as well as a guest account system. It
provides the (very simple) CAS protocol for authentication, which is
generally supported by most things to some degree.

At present, it is occasionally running at https://id.57north.org.uk
from my personal server as I'm developing it - I'd be happy to demo it
next Tuesday.

If we're going to use this for SSO, I'd like to tidy the code up a bit
and comment it some more, have it and its LDAP database moved to a
server owned by the space, and then move a non-critical piece of
infrastructure over to using it, before having Hackhub use it.

Any comments welcome. (And yes, I'm aware its LDAP password is in the
source code.)
