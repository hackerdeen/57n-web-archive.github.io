On 29/07/14 13:46, Iain R. Learmonth wrote:
> [...] Having written mass survey tools where lots of connections
> get constructed and torn down, I can say that C is *vastly* more
> efficient than Python and other interpreted languages.

Hi, Iain.

I think you'll find that Python is a semi-compiled language (*.pyc)
and modern Java hot-spot compilers are ~50% native speed these days.
Using Python or Java avoids having to (mis)manage memory allocation
manually.

I spent two years debugging memory leaks in a large C++ image analysis
program that I wrote with a Chinese colleague and his MSc students...

If you want the speed, without having to program in assembly language,
and don't need to be able to read the code then you should use FORTH :-)
